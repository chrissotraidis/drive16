# Drive16 P2 Plan — close the Genesis-feel gap

REWRITTEN 2026-07-17 after user review of the P1 output: "essentially an
Atari game." The measured diagnosis is `docs/2026-07-17-genesis-feel-gap.md`
— five deficits (tile-quantized motion, zero SFX, static single-frame
sprites, default-font UI, no parallax), all traceable to gates that measure
floors and skeletons that embody the same deficits. P2's job is the quality
ceiling, not delivery polish; app-UI integration moves to the end.

Same loop contract as P0/P1. Branch: `p2/2026-07-17`. Local inference only.
After any `app/src` change: `pnpm --dir app build` + contract/intent/watchdog
green.

---

## P2-G0 — Reference feel profiles from real Genesis games

Status: todo

Run real Genesis ROMs headless through Genteel (extending
`capture-reference-run.py`) and extract a measurable feel profile per game:
pixel-velocity histograms, moving-object counts, audio transient density
(SFX events/sec), animation cadence, palette counts, HUD share, scroll-plane
deltas. Sources: ROM(s) the user supplies from their own collection dropped
into `artifacts/reference-roms/` (behavior analysis only, nothing extracted
or committed), plus permissively-licensed homebrew as a fallback. Output:
`docs/genesis-feel-bar.md` with the numeric bar (e.g. "median moving object
travels 1–3 px/frame at 60 Hz; ≥1 audio transient within 100 ms of every
player action; explosions animate ≥4 frames").

Accept:
- [ ] Profiles captured for ≥2 real games; the P1 Missile Command profiled
      with the same tooling shows the gap numerically on every dimension.
- [ ] `docs/genesis-feel-bar.md` written with hard targets adopted by later
      goals.

## P2-G1 — Game-feel engine: per-pixel motion, SFX, animation in the floor

Status: todo

Raise the floor the model copies. Upgrade skeletons (missile-command first,
then pong/asteroids) and the pass prompts to the bar:

- Per-pixel movement at 60 Hz with velocity/acceleration (no `<< 3` grid
  stepping for moving objects), easing on cursor/paddles, hit-pause and
  screen-shake helpers.
- PSG/XGM SFX layer: shot, explosion, impact, UI blip wired to every player
  action (helper module the skeletons share).
- Multi-frame sprites: 4-frame explosion sequence, 2-frame idle/pulse for
  player objects; `SPR_setAnim` in the art-pass crib and skeletons.
- Custom HUD/title font tiles replacing `VDP_drawText` for game-facing text.
- Plane B parallax scroll in at least two skeletons.

Accept:
- [ ] Upgraded missile-command skeleton measured against the feel bar:
      per-pixel motion histogram in range, SFX transient on ≥90% of actions,
      explosion animates ≥4 frames, parallax delta present.
- [ ] Presentation baseline + all existing checks stay green.

## P2-G2 — Feel gates + self-play scoring

Status: todo

Turn the feel bar into enforcement: a probe that plays scripted sessions and
measures motion granularity (fails tile-stepping), SFX-on-action coverage,
animation cadence, and state transitions (attract → play → game over).
Wire into the polish pass gate (blocking for motion/SFX floors) and the
promote flow; publish `playability-score.json` per build.

Accept:
- [ ] Probe discriminates: old P1 build fails the new gates, upgraded
      skeleton passes; scores for all skeletons recorded.
