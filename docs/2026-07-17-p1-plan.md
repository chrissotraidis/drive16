# Drive16 P1 Plan — quality loop state

Successor to `docs/2026-07-16-remediation-plan.md` (P0, merged in PR #2).
P0 made the system fast and safe; P1 makes the games good. Same loop
contract: work goals in order, one goal per iteration, run the acceptance
checks locally (Ollama qwen only for model calls), flip Status, commit with
the goal id.

Benchmark prompt (the user's bar): *a working Missile Command that looks and
sounds like a real Sega Genesis release — modern, good graphics, good music.*

Ground rules:

- Branch: `quality/2026-07-17-p1`. One commit per goal minimum.
- After any `app/src` change: `pnpm --dir app build`; contract + intent +
  watchdog checks must stay green.
- The benchmark runs through the real product path (browser app, AI sprites +
  MML music enabled) so what we audit is what a user gets.

## P1-G1 — Phased build pipeline (implement → art → music → polish)

Status: todo

The single bounded pass forces gameplay, art, and music to compete for one
20-step/8k-token budget at temp 0. Split the in-app build into sequential
bounded passes over the same project folder: implement+verify (temp 0), art
pass (generate/wire sprites, temp >0), music pass (compose/verify, temp >0),
polish pass. Each pass is its own OpenCode session with its own budget;
Drive16 sequences them and stops when a pass fails its gate.

Accept:
- [ ] App-side pass sequencing behind a setting (default on for new games
      when enhancements are enabled).
- [ ] Each pass runs with its own agent config (steps/tokens/temperature).
- [ ] tsc + contract + a live in-app build with sprites+music produce a game
      where art and music passes actually executed (trace evidence).

Accept:

      derived crop sizes from one diffusion call.
- [ ] A generated background/tileset lands in a test project and builds.

## P1-G3 — Music pipeline v2 (templates, dedicated budget, listen loop)

Status: todo

Three of five skeletons ship a 159-byte stub; MML gets 2 attempts inside the
main pass; the gate counts notes, nobody listens. Add: 3–5 verified-good
genre MML templates (corpus + few-shot in the music pass prompt); replace the
stub VGMs in pong/snake/tetris skeletons with real short tracks compiled from
those templates; music pass gets its own compile budget (≥4 attempts) plus a
render→feature-check→revise-once loop.

Accept:
- [ ] Template library compiles clean with `quality.pass` true for each.
- [ ] Skeleton stub VGMs replaced; presentation baseline still passes.
- [ ] Music pass in P1-G1 uses templates as few-shot and survives 2 syntax
      errors without abandoning music.
